
Cloudtrail : Contineously logs your AWS account activity.

--> Enabled by default. It can store all logs for 90 days without any addl configuration.
--> We cna send logs to S3 bucket, We do have option to send it to cloudwatch and configure retention period / life cycle management rule.
--> We cna filter logs by using multiple options i.e; username, accesskey, eventname, event source..

With CT, we can log only aws activities, not inside ec2 instance / s3 object / lambda data events.. 

4 types of trails available:

1. Management trail : captures aws operations performed on aws console. (instance launch / s3 delete / rds create, iam user creation / delete)
2. Data trail : s3, lambda and dynamodb : Enbale data logs for current & future buckets.
3. Insight events : helps to identify abonormal traffic , errors or user behavious.. 
4. Network trails : Network flow events.. 

** We can enable logs at Organisations level.. 
Organisation: we can combine multiple aws accounts into one umbrilla.

AWS WAF (Web Application Firewall) : Protect your web applications from common web exploits
AWS WAF is a web application firewall service that lets you monitor web requests forwarded to your AWS resources. You can protect CloudFront distributions globally, or regional resources including API Gateway APIs, Application Load Balancers, AppSync GraphQL APIs, Cognito user pools, App Runner services, Verified Access instances, and AWS Amplify applications. Protection is based on conditions you specify, such as originating IP addresses and request patterns.

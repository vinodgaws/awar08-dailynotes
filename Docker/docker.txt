Single application, We perform single deployment for a small change also

Microservices: 
Multiple small services for every module
Services can Scale independently
Independent deployments per service
loosely coupled
We have flexibility to use multiple codebases with seperate programing languages. 
(Notifications --> python, order --> .net, user --> java)
More complex to manage
For every service, we create and use a pipeline.

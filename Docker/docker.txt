Microservices: 
Multiple small services for every module
Services can Scale independently
Independent deployments per service
loosely coupled
We have flexibility to use multiple codebases with seperate programing languages. 
(Notifications --> python, order --> .net, user --> java)
More complex to manage
For every service, we create and use a pipeline.

docker images				--> Lists the local images
docker pull <image>			--> Pulls the docker image from dockerhub to local

docker search <image>
docker search <httpd> --filter is-official=true

docker rmi <image>			--> Removes the docker image from local

docker ps					--> Shows the running containers

docker ps -a				--> List all the containers (including stopped)

docker search httpd --filter is-official=true

docker run <image>			--> RUns the docker image in current terminal (interactive mode)

docker run -d <image>		--> Runs the container in background / detached mode

docker run -d --name my-web <image>		--> Run container in detached mode and a defined name

docker run -it ubuntu /bin/bash			--> Interactively connect to contianer and work on it

docker run -it ubuntu sleep 60

docker run mysql						--> THis will fail as required variable in not passed to container


#Pass env variables while running the container
docker run -d --name mysql-test -e MYSQL_ROOT_PASSWORD=s3cret123 mysql

---
#we can use env file to pass env varibales
docker run -d --name mysql-env --env-file ./myenv.txt mysql

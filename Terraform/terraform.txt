Initilise the Terraform : terraform init

--> Prepares our working directory/folder for terraform usage by initilising it. 
--> It downloads the provider pluings automatically.
--> Validates configuration file structure.

** THis need to run only once when we create a project. (We need to run again if we change the backend)

Resources : We can define what we want to create in AWS environemnt, We need to define using "resource" block.

Resource block contains below 3 imp components:

resource-type: aws_instance 	(should match tf preconfigured names)
Name : webserver				(This can be anything)
Arguments : AMI ID, Instance type

========

terraform statefile : act as source of truth : THis track our resources and its current state. 
** Golden Rule : Never edit this file manually.

terraform fmt				: Formats the existing terraform code with proper indentation
terraform validate			: Looks for systax and configuration errors in tf files.

** Best Practice : Run "fmt" and "validate" before executing the actual commands i.e; apply

terraform plan				: THis is like a dryrun. Tells us what happens if we run this using "terraform apply" option

terraform apply				: THis command actually creates the resources.

terraform apply --auto-approve	:	It wont ask for confirmation to type "yes"

terraform destroy			: Deletes/terminates the resources from our env.

---

Version constraint is recommended method to maintain same version across the project. 

terraform {
  required_providers {
    aws = {
      source = "hashicorp/aws"
      version = "6.48.0"
    }
  }
}

---

= 6.48.0			Exactl version 6.48.0
>= 6.48				Version 6.48 or newer version
<=6.48				Version 6.48 or older version
~> 6.0				Any 6.x version (6.1, 6.2, 6.3) but not 7.x
~> 6.48.0			Any 6.48.x version (6.48.0, 6.48.1, 6.48.2) but not 6.49 / 7

pesimistic constraint : ~

terraform init -upgrade			--> TO upgrade from one version to another version

** Golden Rule : We always specify required_providers at enterprise level, We never depends on "latest".
Mostly we use ~> 
we commit .terraform.lock.hcl file to github to maintain same version lock for the whole team members.

TRACE : Everything - every api call and its response + debug + info + warning + errors
DEBUG : API Calls + info + errors + warning
INFO : General operations info + warnings + errors 
WARN : Warnings + Errors : TO identify potential problems.
ERROR : Errors only : Production / specifically errors monitoring.

To set log level: 

export TF_LOG=INFO
export TF_LOG=DEBUG

export TF_LOG_PATH="tflogs.log"		--> Send all the logs to a dedicated log file. 

unset TF_LOG						--> Turn off logging

	type = string

Number : used for counts, size, port number, timeout..

	type = number
	default = 2
}

Bool (Boolean) : we will have true or false. 
enable_stop_protection : true

variable "enable_stop_protection" {
	type = bool
	default = true
}

---

list(string): an ordered list of strings.. 

variable "instance_type" {
	type = list(string)
	default = ["t3.micro", "t3.small", "t3.medium"]
}

Access : var.instance_type[1]

---

map(string): key value pairs of strings.. 

variable "instance_type_per_env" {
	type = map(string)
	default = [
	dev = "t3.micro" 
	uat = "t3.small" 
	prd = "t3.medium"]
}

Access : var.instance_type_per_env["dev"]

---

tuple : An ordered collection of mixed types - 

variable "server_config" {
	type = tuple([string, number, bool])
	default = ["t3.micro", 22, true]
}

Key Rules for Modules:

Provider block must be in the root module only
Each module (root/child) needs "variables.tf"
run "terrform init" after adding or changing the module.
source argument points to the child module from root module.

github actions: GHA is a CICD platform build on github. THis can track changes/activities happening on our code and perform the automated job. 

--> It has native github integration

---


workflow : A job to automate our processes, We define process in a single YAML file. THis file should be in this location : repo --> .github --> workflows --> xyz.yaml

---

event : An event is the trogger that starts our workflow. 


push				: When any code changes pushed to central repo.
pull request		: When any PR is created.
schedule			: Run at a specific time using cron
workflow_dispatch	: Manually trigger by user

---

Job : A job contains multiple steps that runs on a runner. 

job 
1. verifying code quality			--> step
2. send alert						--> step
3. print result						--> step

Step : A single task inside a job. 

---

Runner : This is a virtual machine that executes our jobs. These are micro VMs (small vm/container)

GH Hosted runners : ubuntu, windows, macos
self-hosted runners : we can configure our server / ec2 to perform the job, its called self hosted runner. 

https://github.com/actions/runner-images

---

Actions : Its a reusable code that does a specific task (i.e; configuring nodejs, configuring aws credentials).. Instead of performing manully with multiple steps, you can depend on this.


actions/checkout@v7			--> checkout repo code
actions/setup-node@v4		--> setup nodejs

conditional workflow demo
----------------
name: Conditional Workflow

on:
    push:
        branches: ["*"]

jobs:
  run-always:
    runs-on: ubuntu-24.04
    steps:
        - name: Print my name
          run: echo "This job is running always"

  only-on-main:
    runs-on: ubuntu-24.04
    if: github.ref == 'refs/heads/main'
    steps:
        - name: Print my name
          run: echo "This job is running only on main branch"

name: Node Js Setup

on:
  push:
    branches: ["*"]

jobs:
    node-setup:
        runs-on: ubuntu-latest
        steps:
            - uses: actions/checkout@v7
            - uses: actions/setup-node@v7
              with:
                node-version: '22'
            - run: npm install
            - run: node index.js

GHA has 3 types of variables:

workflow variables : We can define in our YAML file. 		$VAR_NAME
	--> JOB Level, Workflow Level and Step level
Configuration variables : IN Github UI under settings		${{ vars.VAR_NAME }}
Context variables : prebuild, provided by github			${{ github.VAR_NAME }}

=============================

Github hosted runner : github gives us a VM, THat VM runs our job, Once job done, it Destroys.
--> We can run single job for max of 6 hrs only.

Self-Hosted Runner : You setup your own server/instance, Github sends the job to it, Our machiene runs the job --> Machiene always stays alive.
--> We can access our VPC resources / DBs.
--> we can choose our own HW configs / OS configs
--> we can have pre-installed tools (kubectl, Docker).

OIDC (OpenID Connect) : USing OIDC we can access aws environment resources using an IAM role.. 

Step 1 : Create an Identity provider. Choose "OpenID Connect" as type.

Provider Type: OIDC
Provider URL: https://token.actions.githubusercontent.com
Audience: sts.amazonaws.com

Step 2 : Create an IAM Role for GHA. 

entity type: Web identity
Provider : token.actions.githubusercontent.com
Audience: sts.amazonaws.com
GitHub organization : <yout github account username> : avizway1

Attach required policy: S3readonly role


Step 3 : Create and keep your IAM role.
